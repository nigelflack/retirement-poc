## Design decisions

### Explorer-first app state

`App.jsx` now keeps a single `scenario` object and always renders the scenario explorer. If no scenario exists, the explorer shows a load-first state.

### JSON-only entry path

`ScenarioScreen` supports loading full scenario JSON files that match the current API contract (`pots`, `primaryPot`, `people`, schedules and strategies).

### Monthly income control behavior

The income control updates the first `expenseSchedule` item (`annualAmount`) in the simulation payload. If the scenario has no `expenseSchedule`, a default core-spending entry is synthesized.

### Detail data source

Detail visualizations use `result.resolvedYears` and derive values from yearly `income`, `expense`, `capitalIn`, and `capitalOut` entries.

## Post-completion changes (same session)

The following were made during scenario modelling after v0.18 was marked complete.

### Scenario loader auto-discovery

`ui/src/pages/ScenariosPage.jsx` switched from hardcoded imports to `import.meta.glob('@/scenarios/*.json', { eager: true })`. Any JSON file added to `ui/src/scenarios/` is now automatically listed on the scenarios page.

### New scenario files

| File | Description |
|---|---|
| `ui/src/scenarios/nigel-mimi-same-house.json` | Baseline: stay in current flat, £50k/yr mortgage, 10 years |
| `ui/src/scenarios/nigel-mimi-new-house.json` | House upgrade to £850k, £75k ISA drawdown (£37.5k each), £565k mortgage at £51,314/yr over 15 years |

### Investment return assumption lowered

`server/config/simulation.json` investment mean return changed from **7% → 6%** to better reflect a diversified portfolio in current conditions.
