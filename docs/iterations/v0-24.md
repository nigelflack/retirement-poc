# v0.24 — Income Statement per year

**Status:** complete

---

## Goal

Replace the current flat cashflow detail panel on the Year detail tab with a structured **Household Income Statement** — presenting gross income, tax deductions, net income, expenditure, and net surplus/deficit for the selected year.

This is the second of three financial-report-style sections (v0.23 = Balance Sheet, v0.24 = Income Statement, v0.25 = TBD).

---

## What changes

### Server: add `incomeStatementByYear` to response

`resolvedYears` holds nominal cashflow amounts (what the engine actually processed). To keep the income statement in real terms — consistent with the balance sheet — the adapter will build a parallel `incomeStatementByYear` array with each amount deflated by deterministic cumulative inflation `(1 + inflationMean)^(yearIndex+1)`. This is always returned (not debug-only).

`resolvedYears` is unchanged (still debug-only, still nominal).

The server already returns everything needed:
- `resolvedYears[y].income[]` — line items with `id`, `amount`, `taxable`, `owner`
- `resolvedYears[y].expense[]` — includes `income_tax` and `ni` items
- `resolvedYears[y].tax` — total income tax for the year (scalar)
- `resolvedYears[y].ni` — total NI for the year (scalar)
- `resolvedYears[y].capitalIn[]` / `capitalOut[]`

The income statement presents the same data, restructured:
1. **Gross income** — all income line items with amounts
2. **Less: tax & NI** — `income_tax` and `ni` shown as explicit deductions (sourced from `tax`/`ni` fields, not filtered from the expense list — but `income_tax` and `ni` items must be excluded from the expenditure section below to avoid double-counting)
3. **Net income after tax** = gross income − tax − NI
4. **Expenditure** — all expense items excluding `income_tax` and `ni`
5. **Net surplus / deficit** = net income − expenditure

Capital movements (capitalIn/capitalOut) are shown separately below, unchanged from current layout.

## Data notes

### Server — `incomeStatementByYear`

Added to response (always present, not debug-only). Shape:

```json
"incomeStatementByYear": [
  {
    "yearIndex": 0,
    "income": [ { "id": "employment_nigel", "amount": 80000 }, ... ],
    "tax": 18200,
    "ni": 5100,
    "expense": [ { "id": "groceries", "amount": 30000 }, ... ],
    "capitalIn": [],
    "capitalOut": []
  },
  ...
]
```

All amounts are already in real (today's money) terms — income and expense schedule amounts are specified as real in the scenario JSON and the engine treats them as real (multiplying by cumInflation internally). Tax and NI are computed on those real figures. Mortgage payments are pre-divided by cumInflation before injection so are also real-equivalent. No further deflation is applied.

### UI

- Income statement data sourced from `result.incomeStatementByYear[selectedIndex]`.
- Tax and NI rows always shown (zero if not applicable).
- If `incomeStatementByYear` is absent, show "detailed year data unavailable" message.
