# v0.9 — Detailed breakdown and debug table

_Status: Complete_

---

## What's new

### API changes (breaking)

`POST /run` response changes — pre-production so no backward compatibility required.

**`portfolioPercentiles.byAge`** — each entry gains a `real` field alongside the existing `nominal`:
```json
{ "age": 63, "nominal": [p1..p99], "real": [p1..p99] }
```
Real values are the nominal portfolio deflated by cumulative inflation at each year, in today's money.

**`survivalTable`** — changed from 5-year intervals to annual entries, one per year from `householdRetirementAge+1` to `toAge`:
```json
{ "age": 63, "probabilitySolvent": 0.99 }
```
Previously sampled every 5 years; now every year. The UI derives any 5-year subset it needs client-side.

### Server

- Add `realValuesByYear` parallel to `valuesByYear` in the simulation loop; push `combinedPot / cumInflation` each year
- Compute `allPercentiles(realValuesByYear[y])` and include as `real` in each `byAge` entry
- Replace the 5-year-sampled `survivalTable` output with all `drawdownYears` entries from `solventAtYear`

### UI — Panel 1 breakdown toggle

A `▼ Show detailed breakdown` / `▲ Hide detailed breakdown` toggle below the solvency label text. Collapsed by default. Expands inline within Panel 1 below the toggle row. State persists across re-runs.

Expanded content (three sections, separated by dividers):

1. **How likely is your money to last?** — vertical bar chart, one column per year at 5-year intervals (subset of `survivalTable`), y-axis 0–100%, percentage value below each bar.

2. **What you might have when you retire (in today's money)** — horizontal percentile range track from `accumulationSnapshot.real`: p10–p90 as full extent, p25–p75 as highlighted inner band, p50 marked with a tick. Values rounded to nearest £10,000.

3. **State pension** — per-person name, annual amount, and `from age N`. Section omitted if no person has a state pension.

### UI — Panel 1 debug table toggle

A `Show debug table ▼` / `▲ Hide debug table` toggle on the same row as the breakdown toggle, right-aligned. Independent of the breakdown toggle — both can be open simultaneously.

Expanded content — shown below the breakdown section (or directly below the toggle row if breakdown is collapsed):

**Summary line**: `numSimulations`, `probabilityOfRuin`, `householdRetirementAge` + name, `withdrawalRate` — raw API fields.

**Year-by-year table** — one row per year from current age to `toAge`. Columns:

| Year | [Person A] | [Person B] | Phase | [Person A] SP | [Person B] SP | p10 | p50 | p90 |
|------|-----------|-----------|-------|--------------|--------------|-----|-----|-----|

- **Phase**: `A` (accumulating) or `D` (drawdown); retirement year annotated `← retire`
- **SP columns**: `—` before the person's `fromAge`; annual amount in today's money from `fromAge` onward; omitted entirely if a person has no state pension
- **p10/p50/p90**: from `portfolioPercentiles.byAge[n].real` (real terms, unrounded)
- Table rendered in a fixed-height scrollable container (~12 rows visible); header row sticky

### UI — remove "Calculating…" label

Remove the `<p>Calculating…</p>` element that appears below the solvency bar while a run is in flight. The panel opacity and disabled controls already communicate the loading state; the label adds no information and causes a vertical layout jump.

### CLI

`formatter.py` `format_run` currently iterates over every `survivalTable` entry to print the bar chart. With the table now annual, this must subsample for display — filter to every 5th entry (i.e. where `(age - householdRetirementAge) % 5 == 0`) before rendering the chart rows and the final "probability of running out" line.
