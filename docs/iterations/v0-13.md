## What's new

### 1. Terminology: income target → spending target (breaking)

`incomeSchedule` / `monthlyIncomeTarget` were confusing once per-person income streams were added. Renamed throughout for clarity:

| Old key | New key |
|---------|---------|
| `monthlyIncomeTarget` | `monthlySpendingTarget` |
| `incomeSchedule` | `spendingSchedule` |
| `incomeSchedule[].fromYearsFromRetirement` | unchanged |
| `incomeSchedule[].monthlyAmount` | unchanged |

`annualIncomeTarget` in the response (echoed value) is renamed to `annualSpendingTarget`.

The UI state variable `monthlyIncome` / income-related labels are updated to reflect "spending".

### 2. Per-person income streams

Each person gains an optional `incomeStreams` array. These are regular income flows (e.g. rental income, part-time work, consultancy) that offset the amount the household needs to draw from the portfolio each year.

**JSON shape:**
```json
{
  "name": "Nigel",
  "incomeStreams": [
    {
      "label": "BTL rental",
      "fromYearsFromToday": 0,
      "toYearsFromToday": 12,
      "monthlyAmount": 800
    },
    {
      "label": "Part-time consulting",
      "fromYearsFromToday": 3,
      "toYearsFromToday": 8,
      "monthlyAmount": 1500
    }
  ]
}
```

- `fromYearsFromToday` — year the stream starts (inclusive)
- `toYearsFromToday` — optional; year the stream ends (exclusive — not active in that year). If absent, stream continues to the end of the simulation.
- `monthlyAmount` — monthly £ in today's money (adapter multiplies by 12)
- `label` — optional free-text description

**Adapter:** Sums all income streams across all people into a single `otherIncomeByYear` dense array (length = totalYears). Each active year: `value += stream.monthlyAmount * 12`.

**Engine:** In the drawdown loop, subtract `otherIncomeByYear[y] * cumInflation` from `inflatedTarget` before calculating `draw`, alongside state pension. Income streams are also active during the accumulation phase (years before retirement) — the amount offsets any drawdown that would otherwise occur, but since there is no drawdown in accumulation, it reduces the net required contribution. For simplicity in v0.13: income streams only apply during drawdown years (y >= householdRetirementYear). Accumulation-phase income streams are deferred.

Mathematically for each drawdown year:
```
draw = max(0, inflatedSpendingTarget - totalSP - otherIncomeThisYear)
```

Where `otherIncomeThisYear = otherIncomeByYear[y] * cumInflation`.

### 3. Labels on all items

Optional `label` string added to every schedule/event entry. Used for display in the scenario summary panel. No behavioural effect.

| Object | Label added to |
|--------|---------------|
| Scenario (top-level) | `"label": "Base case — retire at 58"` |
| `contributionSchedule[]` | `"label": "Reduced after redundancy"` |
| `spendingSchedule[]` | `"label": "Reduced spending after 70"` |
| `capitalEvents[]` | `"label": "Boat purchase"` |
| `incomeStreams[]` | `"label": "BTL rental"` |

---

## Out of scope / deferred

- UI inputs for `incomeStreams` or `capitalEvents` — JSON-edit workflow only in v0.13
- Income streams during accumulation phase (before retirement) — deferred; only drawdown years in v0.13
- Scenario summary panel — labelled timeline of capital events, income streams, and spending target in the detailed breakdown (deferred to backlog)
- PCLS (pension commencement lump sum) as a named special case — can be expressed as a `capitalEvents` entry with label "PCLS"; no special handling needed
- Contribution growth rate
- Fan chart visualisation
- Tax modelling

## API changes

### Renamed fields (breaking)

| Old | New |
|-----|-----|
| `monthlyIncomeTarget` (request) | `monthlySpendingTarget` |
| `incomeSchedule` (request) | `spendingSchedule` |
| `annualIncomeTarget` (response) | `annualSpendingTarget` |
| `resolvedSchedules.incomeTargetByYear` (debug) | `resolvedSchedules.spendingTargetByYear` |

### New request fields

```json
{
  "people": [
    {
      "incomeStreams": [
        {
          "label": "BTL rental",
          "fromYearsFromToday": 0,
          "toYearsFromToday": 12,
          "monthlyAmount": 800
        }
      ]
    }
  ],
  "label": "Base case"
}
```

### New debug response field

```json
{
  "resolvedSchedules": {
    "otherIncomeByYear": [0, 0, 9600, 9600, ...]
  }
}
```
