# v0.19 — Scenario Navigation + Year Detail Viewer

**Status:** complete

---

## Goal

Improve day-to-day usability of the scenario explorer by:

1. Fixing missing navigation back to the scenario loader.
2. Replacing dense year-by-year inspection with a focused year detail viewer driven by a horizontal year rail and a slim percentile strip.
3. Improving scenario loader ergonomics with quick actions (reload, create new, clone).

This iteration is UI-only. The simulation engine and API contracts are unchanged.

---

## Scope

### In scope

- Add a clear link from the main scenario explorer screen back to `/scenarios`.
- Add scenario-loader actions:
  - `Reload` (refresh available scenario list/content)
  - `Create new` (start from a blank/default scenario template)
  - `Clone` (duplicate a selected scenario into working runtime state)
- Redesign the detail page inspection experience around a selected year:
  - horizontal year rail (clickable year selection)
  - slim percentile strip (p10-p90 band and p50 line) aligned to year selection
  - one detailed breakdown panel for the selected year
- Keep current fan/spending context where useful, but remove dependence on a large table-first workflow.

Route behavior for this iteration:

- `/` should behave as the loader entry path (redirect or render loader directly)
- scenario exploration runs on a dedicated route (e.g. `/scenario`) for clearer navigation semantics

### Out of scope / deferred

- JSON editor, scenario CRUD redesign, DB/SQLite migration
- New solve logic (`/solve/*` endpoints)
- New simulation assumptions (tax, property logic, liabilities)
- Report generation / LLM narration

---

## Design decisions

### Single-year deep inspection over accordion/table expansion

Large expandable lists are avoided. The user selects one year and sees complete detail for that year in one place.

### Combined navigation + distribution context

The horizontal year rail and slim percentile strip are the primary inspection control. They provide year navigation and percentile context without a full-height chart dominating the screen.

### Scenario-first navigation consistency

The primary scenario screen includes an explicit path back to `/scenarios`.

### Loader-first root path

`/` is treated as a stable entry point to scenario selection. Exploration state lives on its own route.

### Scenario actions are runtime-focused

`Create new` and `Clone` are in-memory/runtime actions in v0.19. Persistent scenario storage redesign (JSON editor/DB) remains deferred.

## Acceptance checks

1. From the scenario explorer screen, user can navigate to `/scenarios` without browser back.
2. `/` opens scenario loader (or redirects there) consistently.
3. Scenario loader includes `Reload`, `Create new`, and `Clone` actions.
4. `Create new` opens scenario explorer with a valid default scenario state.
5. `Clone` opens scenario explorer with a copied scenario and modified label (e.g. "(copy)").
6. Detail page shows a horizontal year selector covering all simulated years.
7. Clicking a year updates a single detail panel for that year.
8. Detail panel includes, for the selected year at minimum:
   - income item breakdown
   - expense item breakdown
   - net cashflow for the year
   - capital event activity (`capitalIn` / `capitalOut`)
   - selected-year percentile values (p10/p50/p90 real)
9. Percentile strip visually shows p10-p90 range and p50 trend across years, with selected-year indicator.
10. Existing detail page navigation fallback still works if `/detail` is opened without state.
11. `npm run build` succeeds in `ui/`.
