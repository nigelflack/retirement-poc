# v0.15 — Detailed scenario viewer

_Status: Complete_

---

## Goal

Replace the inline "show detailed breakdown" and "show debug table" toggles inside Panel 1 with a dedicated detail viewer at `/detail`. The viewer presents three tabbed panels:

1. **Year by year** — expanded debug table with all resolved columns
2. **Fan chart** — p10/p25/p50/p75/p90 portfolio percentile bands over time (real, today's money)
3. **Spending sources** — stacked view of what fills the spending target each drawdown year (portfolio draw, state pension, income streams)

---

## What's new

### 1. ScenarioScreen — remove inline sections / add navigation

- Remove the "Show detailed breakdown" toggle and its three sub-sections (solvency bar chart, accumulation percentile track, state pension summary).
- Remove the "Show debug table" toggle and the scrollable table.
- Remove `showBreakdown` and `showDebug` state.
- Add a `[View detailed breakdown →]` text link below the solvency bar (only visible when a result is available).
- ScenarioScreen always requests with `debug: true`, so `lastResult` will include `resolvedSchedules`.
- `[View detailed breakdown →]` navigates to `/detail` via React Router `navigate`, passing state:
  ```js
  {
    result,              // full POST /simulate response (includes resolvedSchedules)
    people,              // current people array
    capitalEvents,       // current capital events array
    retirementAges,      // { [name]: age }
    monthlyIncome,       // number
  }
  ```

### 2. DetailPage (`/detail`)

New page component. Reads all data from `location.state`. If state is absent (direct navigation), shows a "No scenario loaded" message with a link back.

**Header**: household name string + retirement ages + monthly income. Back link `← Back to scenario`.

**Three tabs**: Year by year | Fan chart | Spending sources

#### Tab 1 — Year by year

Full-width scrollable table. All years from `currentAge+1` to `toAge` (age of reference person). Sticky header.

Columns:
| Column | Source |
|--------|--------|
| Cal. year | `currentYear + (age - refPerson.currentAge)` |
| Per-person age | Per person |
| Phase | A (accumulation) / D (drawdown); ← retire annotation at transition |
| Contributions (£/yr) | `resolvedSchedules.contributionByYear[pi][y]` (total across people) |
| Income streams (£/yr) | `resolvedSchedules.otherIncomeByYear[y]` |
| Spending target (£/yr) | `resolvedSchedules.spendingTargetByYear[y]` |
| Capital event (£) | `resolvedSchedules.capitalEventsByYear[y]` — shown if non-zero |
| Per-person SP | active if personAge >= statePension.fromAge |
| p10 portfolio (real) | `portfolioPercentiles.byAge[n].real[9]` |
| p50 portfolio (real) | `portfolioPercentiles.byAge[n].real[49]` |
| p90 portfolio (real) | `portfolioPercentiles.byAge[n].real[89]` |

Note: resolved schedule values are in today's money (annual £), as set by the adapter. They need no conversion for display.

#### Tab 2 — Fan chart

Recharts `ComposedChart` (x = age, y = £ real portfolio value). Two shaded bands + median line:

| Layer | Percentiles | Fill |
|-------|-------------|------|
| Outer band | p10–p90 | light primary |
| Inner band | p25–p75 | medium primary |
| Median line | p50 | solid primary |

- Vertical reference line at `householdRetirementAge`
- x-axis: age labels; y-axis: abbreviated £ (e.g. `£250k`)
- Accumulation years shown in a different fill tone to retirement years (reference line divides them)

#### Tab 3 — Spending sources

Recharts `BarChart` (stacked, horizontal layout — one bar per drawdown year, x = retirement year offset). Covers drawdown years only.

**Stacked bars** (in today's money, monthly):
| Layer | Derivation |
|-------|-----------|
| Portfolio draw | `max(0, spendingTarget − SP − incomeStreams) / 12` |
| State pension | sum of active annual SP amounts → / 12 |
| Income streams | `otherIncomeByYear[y] / 12` |

- Line overlay: spending target (`spendingTargetByYear[y] / 12`)
- Capital events annotated as vertical dashed reference lines with label

All values in today's money (monthly £) — same frame of reference as the main screen controls.
