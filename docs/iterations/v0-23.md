# v0.23 — Household Balance Sheet

**Status:** complete

---

## Goal

Add a **Household Balance Sheet** section to the Detail page — the first of three financial-report-style sections being built across v0.23–v0.25.

The balance sheet answers: "What does the household own and owe at this point in time, and what is the net equity?"

The current "Net-worth split" 3-card widget shows gross liquid / gross non-liquid / total, with no visibility of individual assets, mortgage liabilities, or true equity. This iteration replaces that widget with a structured balance sheet showing each pot individually on the assets side and each mortgage balance on the liabilities side, producing a per-pot net equity figure.

---

## Scope

### In scope

#### 1. Engine: per-pot p50 tracking; drop nominal from netWorthPercentiles

The engine currently returns aggregate liquid/non-liquid totals only. To show individual pot values in the balance sheet, the engine must track each pot's real balance per path and return the median (p50) per pot per year.

Changes to `runPath`:
- Track per-pot real values at each year: `potsByYearReal[y][pi]`
- Return alongside the existing per-year aggregates

Changes to `simulate`:
- Accumulate per-pot real values across all paths: `potValsByYearReal[y][pi][]`
- After all paths, compute the p50 per pot per year
- Return as `potPercentiles: { byYear: [{ yearIndex, byPot: { [potId]: { realP50: number } } }] }`
- **Drop `nominal` arrays from `netWorthPercentiles`** — they are unused by the UI and CLI. `netWorthPercentiles.byAge[y].liquid`, `.nonLiquid`, and `.total` become `{ real: [...99] }` only.

Memory note: 7 pots × 1 × 48 years × 10,000 paths = 3.36M values × 8 bytes = ~27MB additional heap, comparable to what `netWorthPercentiles` already uses.

#### 2. Adapter: mortgage balances in main response (not debug-only)

Mortgage balances are deterministic (same every simulation path). They are currently only emitted in `resolvedYears[y].mortgageBalances` behind the `debug` flag.

Add a top-level `liabilities` field to the main `/simulate` response:

```json
"liabilities": {
  "byYear": [
    {
      "yearIndex": 0,
      "mortgageBalances": { "main_residence": 415000, "btl_tequila_wharf": 237000 }
    },
    ...
  ]
}
```

- `mortgageBalances` values are in today's money (same basis as the adapter's `propertyState` tracking)
- Pots with no mortgage (or zero balance) are omitted from the object
- This is added to the main response regardless of the `debug` flag

#### 3. Balance sheet UI section (Detail page)

Replaces the existing "Net-worth split" 3-card widget. The section is always shown for the selected year.

**Assets** — one row per pot, labelled by pot id, value is `realP50` from `potPercentiles`. Rows grouped: liquid pots first, then non-liquid.

**Liabilities** — one row per pot with a non-zero mortgage balance in `liabilities.byYear[y].mortgageBalances`.

**Net equity** — Total assets − Total liabilities, displayed prominently.

#### 4. Pass `pots` to Detail page

`ScenarioScreen` currently does not pass the scenario's `pots` array in the navigation state. Add it so that the Detail page has the pot definition list (type, owner, etc.) available for labelling and ordering rows.

### Out of scope

- Balance sheet chart (equity over time). Deferred.
- Balance at percentiles other than p50. Deferred.
- Per-pot owner display or grouping. Deferred.

## Input / output changes

### API response additions

`POST /simulate` response gains two new top-level fields:

**`potPercentiles`** (engine change):
```json
"potPercentiles": {
  "byYear": [
    {
      "yearIndex": 0,
      "byPot": {
        "isa_nigel":     { "realP50": 59000 },
        "pension_nigel": { "realP50": 328000 }
      }
    }
  ]
}
```

**`netWorthPercentiles` (breaking change)** — nominal arrays dropped; shape becomes:
```json
"netWorthPercentiles": {
  "byAge": [
    { "age": 53, "yearIndex": 0,
      "liquid":    { "real": [...99 values...] },
      "nonLiquid": { "real": [...99 values...] },
      "total":     { "real": [...99 values...] }
    }
  ]
}
```

**`liabilities`** (adapter change):
```json
"liabilities": {
  "byYear": [
    { "yearIndex": 0, "mortgageBalances": { "main_residence": 415000, "btl_tequila_wharf": 237000 } }
  ]
}
```

### UI state contract
