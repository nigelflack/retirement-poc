# v0.21 — Detail Tabs Restored + Narrative Text Passthrough

**Status:** planned

---

## Goal

Improve explainability by restoring analyst-oriented detail views while keeping the new year-selector experience, and support narrative/report-style text from scenario JSON.

---

## Scope

### In scope

- Restore detail page tabbed navigation with:
  - Year detail (new rail/selected-year view)
  - Year-by-year table
  - Fan chart
  - Spending sources
- Add optional top-level `openingText` on scenario JSON and render it in UI.
- Pass through optional item-level `label` / `description` fields for schedule and capital items so the detail UI can show natural-language labels.

### Out of scope / deferred

- Dedicated report export format/PDF
- LLM-generated narrative text
- New persistence/storage model

---

## Design decisions

### Keep both interaction modes

The year rail remains the focused inspection mode, while table/fan/spending tabs are restored for deeper analysis and comparison.

### Narrative fields are optional and backward compatible

All new text fields are optional. Existing scenario files without narrative fields continue to work unchanged.

### Adapter passthrough, not engine logic changes

Narrative metadata is carried through adapter debug structures and UI rendering only; simulation math remains unchanged.

## Acceptance checks

1. Detail page has 4 tabs: Year detail, Year by year, Fan chart, Spending sources.
2. Year detail tab keeps the existing year rail + slim percentile strip + selected-year panel behavior.
3. Table/fan/spending tabs render correctly from the same simulation result payload.
